## Tables Created

### 1. USERS Table

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY,
    email VARCHAR(255) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    full_name VARCHAR(255) NOT NULL,
    phone VARCHAR(20),
    status VARCHAR(50) NOT NULL DEFAULT 'ACTIVE'
        CHECK (status IN ('ACTIVE', 'INACTIVE', 'DELETED')),
    created_at TIMESTAMP WITH TIME ZONE NOT NULL,
    updated_at TIMESTAMP WITH TIME ZONE NOT NULL
);
```

**Indexes:**
- `idx_users_email` — Fast email lookup (login, registration uniqueness check)
- `idx_users_status` — Query by user state (find active users)
- `idx_users_created_at` — Temporal queries

**Constraints:**
- `UNIQUE (email)` — Invariant: email uniqueness
- `CHECK (status)` — Enforce valid states (ACTIVE, INACTIVE, DELETED)

**Columns:**
- `id` (UUID) — Primary key, generated by domain layer
- `email` — Normalized lowercase for lookups (case-insensitive)
- `password_hash` — BCrypt hashed (never plaintext)
- `full_name` — User profile
- `phone` — Optional contact
- `status` — User state (ACTIVE = can login, INACTIVE = invited, DELETED = soft-deleted)
- `created_at, updated_at` — UTC timestamps

### 2. WORKSPACES Table

```sql
CREATE TABLE workspaces (
    id UUID PRIMARY KEY,
    owner_id UUID NOT NULL,
    name VARCHAR(255) NOT NULL,
    niche VARCHAR(100) NOT NULL,
    tone_settings JSONB,
    status VARCHAR(50) NOT NULL DEFAULT 'ACTIVE'
        CHECK (status IN ('ACTIVE', 'SUSPENDED')),
    created_at TIMESTAMP WITH TIME ZONE NOT NULL,
    updated_at TIMESTAMP WITH TIME ZONE NOT NULL,
    CONSTRAINT fk_workspaces_owner FOREIGN KEY (owner_id) REFERENCES users(id)
);
```

**Indexes:**
- `idx_workspaces_owner_id` — Find workspaces by owner
- `idx_workspaces_status` — Query by workspace state
- `idx_workspaces_niche` — Filter by business domain
- `idx_workspaces_created_at` — Temporal queries
- `idx_workspaces_name_unique` — Ensure name uniqueness

**Constraints:**
- `UNIQUE (name)` — Invariant: workspace name uniqueness
- `FOREIGN KEY (owner_id)` — References users, ON DELETE RESTRICT (protect owner relationship)
- `CHECK (status)` — Valid states: ACTIVE, SUSPENDED

**Columns:**
- `id` (UUID) — Primary key
- `owner_id` — User who owns this workspace (invariant: must have exactly 1 OWNER member)
- `name` — Workspace name (displayed in UI)
- `niche` — Business domain (social-media, landing-page, branding, web-development, etc.)
- `tone_settings` — JSONB: tone customizations (formal, casual, technical)
- `status` — Workspace state (ACTIVE = operational, SUSPENDED = owner paused)

### 3. WORKSPACE_MEMBERS Table

```sql
CREATE TABLE workspace_members (
    id UUID PRIMARY KEY,
    workspace_id UUID NOT NULL,
    user_id UUID NOT NULL,
    role VARCHAR(50) NOT NULL CHECK (role IN ('OWNER', 'ADMIN', 'MEMBER')),
    status VARCHAR(50) NOT NULL DEFAULT 'ACTIVE'
        CHECK (status IN ('ACTIVE', 'INVITED', 'LEFT')),
    joined_at TIMESTAMP WITH TIME ZONE NOT NULL,
    updated_at TIMESTAMP WITH TIME ZONE NOT NULL,
    CONSTRAINT fk_workspace_members_workspace FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE CASCADE,
    CONSTRAINT fk_workspace_members_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    CONSTRAINT uk_workspace_members_unique UNIQUE (workspace_id, user_id)
);
```

**Indexes:**
- `idx_workspace_members_workspace_id` — Find all members in workspace
- `idx_workspace_members_user_id` — Find all workspaces for user
- `idx_workspace_members_role` — Filter by role (for permission checks)
- `idx_workspace_members_status` — Query by membership state
- `idx_workspace_members_joined_at` — Temporal queries
- `idx_workspace_members_owner_check` — **Critical for invariant:** find all active OWNER members (optimized partial index: `WHERE role = 'OWNER' AND status = 'ACTIVE'`)

**Constraints:**
- `UNIQUE (workspace_id, user_id)` — Invariant: user can only appear once per workspace
- `CHECK (role)` — Valid roles: OWNER, ADMIN, MEMBER
- `CHECK (status)` — Valid states: ACTIVE, INVITED, LEFT
- `FOREIGN KEY (workspace_id)` — ON DELETE CASCADE (if workspace deleted, remove members)
- `FOREIGN KEY (user_id)` — ON DELETE CASCADE (if user deleted, remove memberships)

**Columns:**
- `id` (UUID) — Primary key
- `workspace_id` — Workspace being joined
- `user_id` — User joining
- `role` — OWNER (full access), ADMIN (manage proposals/members), MEMBER (read-only)
- `status` — ACTIVE (member), INVITED (pending acceptance), LEFT (historical)
- `joined_at` — When membership started
- `updated_at` — Last update

**INVARIANTS ENFORCED:**
- `idx_workspace_members_owner_check` — Efficiently verify every workspace has >= 1 OWNER

### 4. OUTBOX Table (Event Sourcing)

```sql
CREATE TABLE outbox (
    id UUID PRIMARY KEY,
    aggregate_type VARCHAR(100) NOT NULL,
    aggregate_id UUID NOT NULL,
    event_type VARCHAR(100) NOT NULL,
    event_payload JSONB NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE NOT NULL,
    published_at TIMESTAMP WITH TIME ZONE
);
```

**Indexes:**
- `idx_outbox_published_at` — **Critical for publishing worker:** find unpublished events (`WHERE published_at IS NULL`)
- `idx_outbox_event_type` — Filter by event type (debugging, replay)
- `idx_outbox_created_at` — Temporal queries

**Purpose:**
- Transactional outbox pattern: ensures no event is lost if publishing fails
- Atomically persist domain event + database changes in same transaction
- Publishing worker polls and publishes to Kafka/RabbitMQ
- After successful publish, updates `published_at` timestamp

**Columns:**
- `id` (UUID) — Unique event ID
- `aggregate_type` — Domain aggregate (User, Workspace, WorkspaceMember)
- `aggregate_id` — ID of aggregate that produced event
- `event_type` — Event class name (UserRegistered, WorkspaceMemberInvited)
- `event_payload` — JSON: full event data (userId, email, timestamp, etc.)
- `created_at` — Event creation time
- `published_at` — NULL = not yet published, filled when sent to broker

### 5. ACTIVITY_LOGS Table (Audit Trail)

```sql
CREATE TABLE activity_logs (
    id UUID PRIMARY KEY,
    workspace_id UUID,
    user_id UUID,
    action VARCHAR(100) NOT NULL,
    entity_type VARCHAR(100) NOT NULL,
    entity_id UUID NOT NULL,
    changes JSONB,
    ip_address VARCHAR(45),
    user_agent TEXT,
    created_at TIMESTAMP WITH TIME ZONE NOT NULL,
    CONSTRAINT fk_activity_logs_workspace FOREIGN KEY (workspace_id) REFERENCES workspaces(id) ON DELETE SET NULL,
    CONSTRAINT fk_activity_logs_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL
);
```

**Indexes:**
- `idx_activity_logs_workspace_id` — Find actions in a workspace
- `idx_activity_logs_user_id` — Find actions by user
- `idx_activity_logs_entity` — Find actions on specific entity
- `idx_activity_logs_action` — Filter by action type
- `idx_activity_logs_created_at` — Temporal queries

**Purpose:**
- Immutable audit trail: INSERT ONLY, never UPDATE or DELETE
- Compliance & debugging: full history of who did what and when
- IP address + User-Agent for security forensics

**Columns:**
- `id` (UUID) — Log entry ID
- `workspace_id` — Workspace where action occurred (nullable for system actions)
- `user_id` — User who performed action (nullable for system actions)
- `action` — Action type (created, updated, deleted, invited, left, approved, etc.)
- `entity_type` — Entity affected (User, Workspace, WorkspaceMember, Proposal, etc.)
- `entity_id` — ID of entity affected
- `changes` — JSONB: before/after diff (for debugging)
- `ip_address` — Client IP (IPv4 or IPv6)
- `user_agent` — Browser/client info
- `created_at` — Action timestamp

### 6. Views

#### v_workspace_members_active
```sql
CREATE VIEW v_workspace_members_active AS
SELECT
    wm.id,
    wm.workspace_id,
    wm.user_id,
    u.email,
    u.full_name,
    wm.role,
    wm.joined_at,
    wm.updated_at
FROM workspace_members wm
JOIN users u ON wm.user_id = u.id
WHERE wm.status = 'ACTIVE';
```

**Purpose:** Fast query to get all active members with user details (name, email)

#### v_workspace_owners
```sql
CREATE VIEW v_workspace_owners AS
SELECT
    workspace_id,
    COUNT(*) as owner_count
FROM workspace_members
WHERE role = 'OWNER' AND status = 'ACTIVE'
GROUP BY workspace_id;
```

**Purpose:** Verify invariant (should always have >= 1 owner per workspace)

## Multi-Tenancy & Workspace Isolation

### Isolation by Design

1. **Every query filtered by workspace_id:**
   - Example: "Get all members in workspace X" → `SELECT * FROM workspace_members WHERE workspace_id = ?`
   - Example: "Get activities in workspace X" → `SELECT * FROM activity_logs WHERE workspace_id = ?`

2. **Foreign key constraints enforce hierarchy:**
   - `workspace_members` → `workspaces` → `users`
   - Cascade delete on workspace deletion (clean up members)
   - Restrict delete on user if they own workspaces (protect data integrity)

3. **Application layer enforces:**
   - JWT contains workspace_id → all queries include workspace_id filter
   - Impossible to query across workspaces (tenant isolation)

## Event Sourcing (Outbox Pattern)

### How It Works

1. **User registers:**
   ```
   BEGIN TRANSACTION
     INSERT INTO users (id, email, password_hash, ...) VALUES (...)
     INSERT INTO outbox (aggregate_type, aggregate_id, event_type, event_payload)
       VALUES ('User', user_id, 'UserRegistered', JSON_PAYLOAD)
   COMMIT
   ```

2. **Publishing worker polls:**
   ```
   SELECT * FROM outbox WHERE published_at IS NULL ORDER BY created_at LIMIT 100
   ```

3. **For each event:**
   ```
   PUBLISH event_payload to Kafka/RabbitMQ
   UPDATE outbox SET published_at = NOW() WHERE id = ?
   ```

4. **Guarantees:**
   - ✅ No event is lost (if broker fails, retry later)
   - ✅ Exactly-once semantics (publishing worker idempotent)
   - ✅ Other bounded contexts (Briefing, Proposal) subscribe and react
